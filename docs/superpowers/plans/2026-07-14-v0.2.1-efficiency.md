# Microsoft Excel BI Agent v0.2.1 Efficiency Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce plugin discovery cost and improve skill-trigger precision without changing the twelve published skill IDs or adding Excel functionality.

**Architecture:** Add a deterministic trigger-contract validator and customer-data-free benchmark corpus before changing metadata. Then shorten the manifest prompts and canonical skill descriptions, sync generated mirrors, measure the staged runtime package before and after, and publish the change as `v0.2.1` through a draft pull request.

**Tech Stack:** Python 3 standard library, JSON, Markdown skills, plugin-eval CLI, GitHub Actions, Git.

## Global Constraints

- Keep all twelve published skill IDs unchanged.
- Edit only `.agents/skills/`; generate `skills/`, `.claude/skills/`, and `.opencode/skills/` with `tools/sync-skills.py`.
- Every canonical description starts with `Use when`, contains trigger context rather than workflow steps, and is at most 300 characters; target 120 to 220 characters.
- Keep exactly three manifest starter prompts, each at most 110 characters.
- Preserve existing Excel routing and compatibility behavior.
- Do not add Excel, Power BI, VBA, Power Query, DAX, MDX, or ADO features.
- Synthetic benchmark output validates mechanics only and is never presented as real task-success evidence.
- Keep local plugin-eval runs, observed usage logs, and machine-specific reports out of Git.
