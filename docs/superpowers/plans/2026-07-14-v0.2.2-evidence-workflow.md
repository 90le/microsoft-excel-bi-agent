# v0.2.2 evidence workflow implementation plan

> **For Codex:** execute this plan with `superpowers:subagent-driven-development`.

**Goal:** Add a privacy-safe, deterministic evidence workflow: five regression fixtures, local observed-usage validation and summarization, and release-gate/documentation integration.

**Architecture:** Checked-in cases remain synthetic JSON fixtures. Local observed usage is JSONL, validated before summarization, never auto-collected or uploaded, and must never contain customer files, paths, credentials, or workbook data. The release gate verifies repository contracts only; it never requires Excel or local usage logs.

**Global constraints**

- Do not commit customer workbooks, real logs, filesystem paths, credentials, or copied data.
- Observed usage remains opt-in, local-only JSONL; no telemetry, upload, network client, or automatic collection.
- `workbook-behavior` evidence is valid only for `local-user-supplied-sanitized` cases.
- Validator rejects absolute/relative paths, credentials, and customer artifact names in every string field.
- Preserve the existing seven fixture cases and increase the manifest to exactly twelve cases.
- Do not change the current stable release/version; this is an unreleased branch implementation.

## Task 2: Add the local observed-usage validator

**Files:**
- Create: `tools/validate_observed_usage.py`
- Create: `tests/test_observed_usage.py`
- Modify: `.gitignore`

**Step 1: Write failing tests**

Test a valid JSONL event and invalid events for: malformed JSON, missing required field, unsupported enum, negative duration, duplicate event ID, invalid timestamp, forbidden path, credential-like value, customer artifact name, and invalid workbook-behavior case pairing.

**Step 2: Implement `validate_observed_usage.py`**

Provide a CLI accepting one or more JSONL files. Validate these required fields:

`schemaVersion` (`1.0`), `eventId`, `recordedAt` (UTC ISO-8601), `caseId`, `requestedSkill`, `selectedSkill`, `outcome`, `durationMs` (nonnegative integer), `evidenceLevel`, and `evidenceBoundary`.

Allowed `evidenceLevel` values: `structural`, `runtime-capability`, `workbook-behavior`. Allow only `local-user-supplied-sanitized` for workbook behavior. Reject strings containing path-like content, credentials or customer artifact names. Print actionable file/line/field diagnostics and exit nonzero for invalid input. Do not read workbooks or access the network.

**Step 3: Ignore local usage data**

Ignore a repository-local usage folder such as `.local-observed-usage/` while retaining no real usage data in Git.

**Step 4: Verify**

Run:

