Excel BI Agent Skill Pack

让 AI Agent 真正会处理复杂 Excel BI 工作簿

Microsoft Excel BI Agent Pack 是一套本地插件和 skills 包。它不是一个普通提示词集合，而是把 Excel/VBA、Power Query M、Power Pivot DAX、MDX/CUBE、ADO/SQL、交付清理和 QA 验证 做成可安装、可复用、可验证的 agent 工作体系。

这个项目可以干嘛

它的目标是让不同 AI agent 在处理 Excel BI 文件时，不再只靠临时猜测，而是按稳定流程完成读取、定位、修改、验证和交付。

Workbook

改 Excel 工作簿

创建、修改、检查 `.xlsx` / `.xlsm`，处理公式、格式、隐藏 sheet、按钮、控件、保护、外链和交付结构。

VBA

调试 VBA

导出/导入模块，定位编译错误、运行时错误、按钮绑定问题，并保持宏工作簿结构不被破坏。

Power Query

处理 M 查询

检查、编辑、插入、删除 Power Query，分析依赖、刷新状态、错误分类、性能耗时和数据源风险。

DAX / Model

分析 Power Pivot

处理 Data Model、DAX 度量、计算列、关系、上下文、兼容性和 Excel Power Pivot 与 Power BI 边界。

MDX / CUBE

追踪 CUBE 公式

分析 CUBEVALUE、CUBEMEMBER、MDX 成员、度量引用、动态期间和报表层公式依赖。

ADO / SQL

连接 Excel 数据

支持 ADO、OLEDB、ADODB、ADOMD、SQL 查询工作簿、外部文件和可访问的数据模型接口。

Delivery

生成纯净交付物

冻结公式为值，删除外链、Power Query、Data Model、配置 sheet 和过程稿，输出客户可打开的干净版本。

QA

做交付前审计

检查公式质量、外部依赖、控件、隐藏内容、视觉可读性、发布门禁和客户交付风险。

为什么需要它

减少临时猜测

Excel BI 文件通常同时包含公式、宏、查询、模型和外链。这个包让 agent 先选正确层，再操作。

降低交付风险

交付前清理、验证、审计和边界说明固化为流程，避免把过程稿、外链或私有路径发给客户。

多 agent 一致

Codex、Claude、OpenCode 可以同步同一套 skills，减少不同 agent 对同一任务的处理方式漂移。

Windows 安装命令

python tools\deploy-local-plugin.py --project-root . --replace --install
python tools\sync-skills.py --project-root . --all-project-mirrors --codex-user --replace
python tools\run_release_gate.py --project-root . --profile structural

macOS / Linux / Git Bash

python tools/deploy-local-plugin.py --project-root . --replace --install
python tools/sync-skills.py --project-root . --all-project-mirrors --codex-user --replace
python tools/run_release_gate.py --project-root . --profile structural

完整 Excel COM、VBA 执行、Power Query 刷新和 Power Pivot/Data Model 运行验证需要 Windows + 桌面版 Microsoft Excel。
