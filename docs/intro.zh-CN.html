Excel BI 工作簿不只是代码。

真实工作簿可能同时包含公式、VBA、Power Query M、Power Pivot 度量值、CUBE 引用、名称管理器、外部链接、按钮、隐藏过程表和交付副本。这个项目把这些风险沉淀成可复用的 Agent 工作流。

更安全地修改工作簿

为公式、VBA、按钮、隐藏表、工作簿链接和结构保护提供明确操作流程。

理解 Excel BI 层

为 Power Query、Power Pivot DAX、MDX/CUBE、ADO/SQL 和语义模型边界提供专项技能。

面向客户交付

支持公式转值、删除依赖、清理过程表，并验证可交付的纯净 Excel 文件。

定位损坏的 BI 层

把 Power Query、DAX、MDX/CUBE、ADO/SQL 或工作簿结构问题路由到正确专项工作流。

区分结构和运行时

发布更干净的包

在发布面向工作簿的技能包前，使用 artifact hygiene、安装检查和 release 可见性。

12 个面向生产场景的 Excel BI 技能。

宽泛需求先使用路由技能；当工作簿层级明确时，直接进入对应专项技能。

VBA 与工作簿工程
excel-vba-workbook-engineering 用于导出/导入模块、检查按钮、保护公式结构，并在可用环境中验证 Excel 行为。
Power Query M
power-query-m-engineering 处理查询编辑、刷新完成等待、依赖血缘、错误定位和性能计时。
Power Pivot / DAX / MDX
power-pivot-dax-modeling 与 mdx-cubevalue-extraction 覆盖度量值、上下文、成员、CUBE 公式和数据模型边界。
ADO / SQL 访问
excel-ado-sql-data-access 覆盖工作簿 SQL、ADODB、ADOMD、Provider 字符串、参数和 Recordset 处理。
QA 与交付
excel-workbook-qa-auditor 与 excel-deliverable-publisher 把风险审查和纯净 `.xlsx` 发布变成可复用流程。
跨 Agent 分发
从同一份 .agents/skills/ 技能源生成 Codex、Claude、OpenCode 镜像。

安装一次，多 Agent 复用。

Codex 用户优先使用插件市场；需要审查或定制的团队使用本地一键安装；企业内部分发可以使用显式脚本安装和同步。
